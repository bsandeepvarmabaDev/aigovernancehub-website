Sample files & validation guide

Use these samples to verify your export format before uploading real project data. Every assessment validates your file server-side — samples help you get it right the first time.

Jira Cloud

Download Jira sample CSV View example report

Required columns

Issue Key, Summary, Description, Issue Type, Status, Project

Optional columns (recommended)

Priority, Labels, Components, Assignee, Reporter, Story Points, Sprint

How to export

  1. Open your Jira Cloud project
  2. Issues → Search for issues (JQL) or backlog
  3. Export → CSV (All fields)
  4. Upload in the assessment wizard

Validation expectations

UTF-8 CSV up to 5 MB. At least one data row. Duplicate Issue Keys are flagged. Work items counted server-side for plan detection.

  3. Upload in the assessment wizard

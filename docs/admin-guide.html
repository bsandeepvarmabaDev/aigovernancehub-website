Administration Guide

This guide is for Jira administrators, governance leads, and app owners responsible for operating AI Governance Hub.

Admin Responsibilities

  • Confirm authorized users and reviewers.
  • Define governance review expectations.
  • Maintain workflow consistency.
  • Review app configuration and audit activity.
  • Ensure sensitive information is not entered where it is not required.

Role-Based Governance

Organizations should map governance actions to appropriate roles such as Admin, Reviewer, Governance Lead, Security Reviewer, Privacy Reviewer, Responsible AI Reviewer, Legal/Compliance Reviewer, and Business Owner.

Review Operations

Use Jira issues and AI Governance Hub review views to track risk, evidence, decisions, comments, approval progress, and follow-up actions.

Security Practices

  • Use Jira permissions and groups appropriately.
  • Do not enter passwords, personal access tokens, or shared secrets into governance notes.
  • Review security findings through official support and security channels.
